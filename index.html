Step by Step

Step 1 — Pick your tools (you can mix & match)

📧
Email: Gmail or Outlook
📁
Files: Google Drive, Dropbox, or OneDrive
🗒️
Notes: Notion, Evernote, or Google Docs
🤖
AI: OpenAI or your favorite model
🗓️
Calendar: Google or Microsoft
⚙️
Automation: Zapier, IFTTT, Make, or n8n

Don’t worry. Any mix is okay. We will show recipes for each.

Step 2 — Make one folder to watch

Why? Put new files here (PDFs, images). Your robot will look in this folder.

  • Create folder named /INBOX
  • Drop a PDF or screenshot inside
  • We will read it later

Bonus: Make an email label “Newsletters”. Move interesting emails there.

  • Gmail/Outlook: create label/folder
  • Move 2 emails into it

Step 3 — Choose your automation tool

Zap 1 — Get things in
  • Trigger: New email in label “Newsletters”
  • Action: Add to Notes (Notion/Docs) as “Inbox Item”
  • Action: If attachment → send to OCR step
Zap 2 — Read & Summarize
  • Trigger: New item in “Inbox Items”
  • Action: OpenAI — make 5 short bullets + tags
  • Action: Update the “Inbox Item” with summary + tags
Zap 3 — Sort & Decide
  • Trigger: Item updated with summary
  • Action: Paths (rules): if “invoice” → P0 + Reply; if “flight < $250” → Notify; else P2
Zap 4 — Do the thing
  • If Reply → send email draft
  • If Schedule → add event to Calendar
  • If Draft → create Doc + link it
  • If File → move file in Drive
  • If Notify → email/text yourself
Applet A — Inbox
  • If email in “Newsletters” → add to Notes
  • If file in /INBOX → run OCR
Applet B — Summaries
  • If new Note → call AI (Webhook) → get 5 bullets
  • Then update the Note with tags
Applet C — Actions
  • If tag “P0” → send quick alert
  • If tag “Reply” → draft email
Applet D — Brief
  • Every day at 9am → build a daily note
  • Add top 5 and urgent items
Scenario 1 — Ingest
  • Watch Gmail label + Watch Drive folder
  • Route PDFs to OCR → text
  • Create/Update item in Notion/Docs
Scenario 2 — Summarize
  • OpenAI/HTTP — summarize to 5 bullets + tags
  • Write tags & priority back
Scenario 3 — Decide
  • Routers/Filters: if “invoice” → P0 + Reply
  • Else if “launch” → P1 + Draft
Scenario 4 — Act
  • Send email / create event / move file
  • Log to a “Log Book” table

  • IMAP Email → If attachment → OCR
  • Drive Trigger → OCR → Add to Notion
Flow B — Summarize & Tag
  • OpenAI node → 5 bullets + entities
  • Update record in Notion
Flow C — Decide
  • IF nodes for rules (P0/P1/P2)
  • Optional: second AI check
Flow D — Actions
  • Send email / calendar / doc / move file / notify
  • Write to “Log Book” database

Step 4 — Make your Daily Brief (9:00 AM)

What: A tiny morning report with: 5 highlights, urgent items, calendar changes, top links.

  • Schedule: every day at 9:00 AM
  • Send to: your email (and phone if you want)

Prompt:

Make a morning brief with: 5 highlights, decisions needed (P0),
schedule changes, and top links. Be short. Use bullets.

Step 5 — Make your Inbox with Buttons

Simple way: Use a Kanban board in Notion, Trello, or your notes app.

  • Columns: P0 (now), P1 (soon), P2 (later)
  • Each card links back to the email or file
  • Add buttons: Approve, Snooze, Archive

How the buttons work:

  • Button → calls a Webhook
  • Webhook → runs a tiny automation
  • Automation → does the thing + logs it

Step 6 — Keep a Log Book

Why: So you can see what happened and undo things.

  • Make a table: When, What, Item, Result, Details
  • Write one row for every action
  • Export anytime
Privacy Tip: Give each tool only the smallest permissions it needs. Read‑only is best when you can.

You did it! 🎉 What now?

  • Turn on “Notify” first (easy wins)
  • Next, turn on “Draft” and “Schedule”
  • Last, add your “Inbox with Buttons” board

Want more power later? Add extra rules like “If price < $250 → alert” or “If ‘launch’ → create social post.”
